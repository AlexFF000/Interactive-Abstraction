What is 32 bit mode?By default, the processor uses 8 bit buses and registers. This allows for a maximum of 256 (2^8) memory addresses, which is too few for more complex programs. 32 bit mode increases the address bus and some registers to 32 bits, allowing a maximum of 4,294,967,296 (2^32) addresses, therefore providing space for more complex programs. Note that 32 bit mode only allows for more memory addresses, it does not allow the processor to handle data that is more than 8 bits.
Which buses and registers are different in 32 bit mode?The Address Bus, Program Counter, Current Instruction Register, and Memory Data Register are all 32 bits wide. The other buses and registers remain 8 bits wide.
Are the instructions different in 32 bit mode?The machine code instructions follow a slightly different format in 32 bit mode. The instruction must specify the number of memory addresses used by the operand. Therefore the new format is as follows: opcode (4 bits) addressing mode (1 bit) number of addresses needed to store operand (3 bits) operand (8-32 bits). If using assembly code then the converter will work out the operand length automatically, meaning that the format for assembly code remains unchanged (aside from the fact that addresses larger that 256 can be used).
Drawbacks of using 32 bit modeThe 4 billion memory addresses allowed by 32 bit mode is far too many to display, therefore the memory table UI is replaced with a more limited interface which allows the user to specify a small number of addresses to be displayed. Another drawback is that the databus (which remains 8 bits) is a bottleneck when transferring data between 32 bit registers, which means that the data must be transferred in 4 parts. This adds a significant number of extra clock cycles to each instruction, so the processor is slower when in 32 bit mode.
